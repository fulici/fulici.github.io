【教程】Hugo+Github免费部署自己的博客

1.环境准备

1.1Git下载

  • 前往【Git官网】,下载安装程序
  • 一直点下一步，默认安装即可

1.2Hugo下载

  • 前往【 [Hugo Github Tags](Tags · gohugoio/hugo (github.com))】，选择对应版本下载，下载后解压即可
  • Windows下载版本：hugo_extended_xxxxx_windows_amd64.zip



2.搭建博客

  • 打开解压文件，在文件夹上面输入cmd 进入命令行，输入 hugo new site dev 创建dev 文件夹

  • 把hugo.exe 复制添加到刚刚创建的dev 文件夹

  • 命令行中输入cd dev 进入dev文件夹，输入 hugo server -D 启动博客

  • 在浏览器中输入http://localhost:1313/ ,进入博客，显示如下：



    此时，服务已经搭好了。

  • 进入Hugo官网，下载一个自己喜欢的主题（这里选择“Stack”主题）

  • 跳转到Github 网站，点击Tags ,选择一个版本，下载Source code(zip)





  • 下载后解压，把解压后的文件放到themes 文件夹中

  • 把主题文件夹exampleSite（样例文件） 文件中的content 和hugo.yaml 文件复制一份放到dev 文件夹中



  • 复制后，删除hugo.toml 文件，两个文件作用 （配置文件） 一样，只是后缀不一样，删除一个。



  • 删除dev/content/post中的rich-content 文件，rich-content 文件中引用了YouTube 的视频，可能会访问超时，导致项目无法启动



  • 用记事本打开配置文件hugo.yaml ，进行配置。【参考博客配置文件】

  • 文章位置：dev/content/post ，其中的文件夹名就是文章名。



  • 修改文章编码：en-英文 ，没有后缀为默认



  • 创建一篇新文章，命令行输入 hugo new content post/myFirstBlog/index.md ,然后重启项目，刷新博客界面

  • 更改左边的英文为中文，dev/content/page ,然后逐个打开markdown文档进行修改

3.Github常规部署

  • 在GitHub 上创建仓库，仓库名为： fulici.github.io

  • 在本地更改 hugo.yaml 配置文件，更改baseurl: https://fulici.github.io

  • 删除public 文件，在命令行中输入 hugo -D 重新生产 public 文件，此时， public 文件中的内容即为要上传的网页内容

  • 然后在 public 文件目录下，命令行依次输入以下命令:

    • git init

    • git add .

    • git commit -m "first commit"

    • git branch -M main

    • git remote add origin https://github.com/fulici/fulici.github.io.git

    • git push -u origin main

  • 开启静态页面，Seetings->Pages ，选择main 分支，在此页面显示https://fulici.github.io/ 时，点击这个链接即可看到部署好的博客

4.Github自动部署

  • 在GitHub 上创建一个新仓库 hugo-main 用于存储hugo主文件 ，仓库设置为私有

  • 在hugo主文件 中创建 .gitignore ，用于忽略上传文件 ：public ，resources ，.hugo build.lock ，hugo.exe

  • 然后在dev 文件目录下，命令行依次输入以下命令：

    • git init
    • git add .
    • git commit -m "first commit"
    • git branch -M main
    • git remote add origin https://github.com/fulici/hugo-main.git
    • git push -u origin main

  • 获取TOKEN Settings -> Secrets and variables -> Actions

  • 回到电脑本地的dev主文件 ，创建一个.github/workflows/xxxx.yaml 文件，将以下内容复制进去 name: deploy

    代码提交到main分支时触发github action

    on:

    push:
  branches:
    - main

    jobs:

    deploy:
  runs-on: ubuntu-latest
  steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
            fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
            hugo-version: "latest"
            extended: true

      - name: Build Web
        run: hugo -D

      - name: Deploy Web
        uses: peaceiris/actions-gh-pages@v4
        with:
            PERSONAL_TOKEN: ${{ secrets.你的token变量名 }}
            EXTERNAL_REPOSITORY: 你的github名/你的仓库名
            PUBLISH_BRANCH: main
            PUBLISH_DIR: ./public
            commit_message: auto deploy

  • 创建一个新文章进行测试，命令行输入 hugo new content post/自动部署测试/index.md

  • 然后命令行输入hugo server -D 启动博客，先在本地查看刚刚的博客是否显示

  • 然后push代码 ，命令行依次输入：

    • git add .

    • git commit -m "update"

    • git push

  • 回到Github的hugo-main 仓库，以下界面出现则部署成功
